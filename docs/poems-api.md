# API Poems

## Общая информация

**Базовый путь:** `/api/poems/`

Все эндпоинты требуют JWT-токен в заголовке `Authorization: Bearer <token>`.

Объект стиха (`Poem`) имеет следующие поля:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `int` | Идентификатор стиха |
| `title` | `string` | Заголовок. Если не задан явно, выводится из первой строки текста в формате `"<Первая строка>..."` |
| `content` | `string` | Текст стиха (нормализуется при сохранении) |
| `comment` | `string` | Комментарий. Если не задан, подставляется текущая дата в формате `ДД.ММ.ГГГГ` |

**Формат ошибок:**

```json
{
  "error": {
    "message": "Текст ошибки",
    "type": "ErrorType",
    "code": 400,
    "form_errors": {
      "field": [
        "Описание ошибки поля"
      ]
    }
  }
}
```

## 1. Список стихов

Возвращает список стихов текущего автора с пагинацией.

**`GET /api/poems/`**

### Параметры запроса (query params)

| Параметр | Тип | Обязательный | По умолчанию | Описание |
|----------|-----|--------------|--------------|----------|
| `collection` | `int` \| `null` | Нет | `null` | Фильтр по коллекции. `null` — стихи вне коллекций. |
| `query` | `string` | Нет | `""` | Поиск по вхождению подстроки в `title`, `content` или `comment`. |

### Ответ `200 OK`

```json
{
  "count": 42,
  "next": "http://host/api/poems/?page=2",
  "previous": null,
  "results": [
    {
      "id": 1,
      "title": "\"Первая строка...\"",
      "content": "текст",
      "comment": "16.07.2026"
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `count` | `int` | Общее количество стихов |
| `next` | `string` \| `null` | URL следующей страницы |
| `previous` | `string` \| `null` | URL предыдущей страницы |
| `results` | `array` | Массив стихов |
| `results[].id` | `int` | Идентификатор стиха |
| `results[].title` | `string` | Заголовок стиха |
| `results[].content` | `string` | Текст стиха |
| `results[].comment` | `string` | Комментарий |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `400` | `InvalidFormError` | Invalid form | Невалидные query-параметры |

## 2. Создание стиха

Создаёт новый стих со `active=True` и без коллекции (`collection=null`). Позиция (`position`) вычисляется как максимальная позиция среди стихов автора вне коллекций + 1.

**`POST /api/poems/`**

### Тело запроса

```json
{
  "content": "Текст стиха.\nВторая строка.",
  "title": "Необязательный заголовок",
  "comment": "Необязательный комментарий"
}
```

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| `content` | `string` | Да | Текст стиха. Пробелы/пустые края нормализуются. |
| `title` | `string` | Нет | Если пусто — выводится из первой строки `content`. |
| `comment` | `string` | Нет | Если пусто — подставляется текущая дата. |

### Ответ `201 Created`

```json
{
  "id": 1,
  "title": "\"Первая строка...\"",
  "content": "Текст стиха.\nВторая строка.",
  "comment": "16.07.2026"
}
```

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `400` | `InvalidFormError` | Invalid form | Невалидные или отсутствующие поля |

## 3. Статистика для дашборда

Статистика по активным стихам (`active=True`) текущего автора.

**`GET /api/poems/stats/`**

### Ответ `200 OK`

```json
{
  "in_collection": 12,
  "uncategorized": 30,
  "remaining": 99958,
  "avg_scores": {
    "freshness": 7.3,
    "emotional_density": 6.8,
    "voice": 7.1,
    "completeness": 6.5,
    "anti_repeat": 9.2
  },
  "count_assessed": 42
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `in_collection` | `int` | Количество активных стихов, привязанных к коллекции (`collection != null`). |
| `uncategorized` | `int` | Количество активных стихов вне коллекций (`collection = null`). |
| `remaining` | `int` | `max(0, 100000 - (in_collection + uncategorized))` — лимит 100 000 стихов. |
| `avg_scores` | `object` | Средние баллы по оценённым активным стихам автора (см. раздел 5). `null`-поля — если ни у одного стиха нет оценки по шкале. |
| `avg_scores.freshness` | `float` \| `null` | Средняя свежесть образа. |
| `avg_scores.emotional_density` | `float` \| `null` | Средняя эмоциональная плотность. |
| `avg_scores.voice` | `float` \| `null` | Средний голос. |
| `avg_scores.completeness` | `float` \| `null` | Средняя завершённость. |
| `avg_scores.anti_repeat` | `float` \| `null` | Средний анти-повтор (вычисляется из эмбеддингов, см. раздел 5). |
| `count_assessed` | `int` | Количество активных стихов с сохранённой оценкой. |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |

## 4. Обновление стиха

Полное обновление стиха. Доступно только автору стиха.

**`PUT /api/poems/{id}/`**

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | `int` | ID стиха |

### Тело запроса

Все поля необязательны (`partial=True`, сервер дополняет `title`/`comment` из `content`, если они не переданы):

```json
{
  "content": "Новый текст стиха.",
  "title": "Новый заголовок",
  "comment": "Новый комментарий"
}
```

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| `content` | `string` | Нет | Если передан — нормализуется. `title`/`comment` пересчитываются, если не заданы явно. |
| `title` | `string` | Нет | Если пусто/не передан — выводится из первой строки `content`. |
| `comment` | `string` | Нет | Если пусто/не передан — подставляется текущая дата. |

### Ответ `200 OK`

```json
{
  "id": 1,
  "title": "Новый заголовок",
  "content": "Новый текст стиха.",
  "comment": "Новый комментарий"
}
```

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `400` | `InvalidFormError` | Invalid form | Невалидные или отсутствующие поля |
| `404` | `PoemNotFoundError` | Poem not found | Стих не найден или не принадлежит текущему автору |

## 5. Оценка стиха

После создания/обновления стиха асинхронно (очередь `celery`) запускается задача
`assess_poem`, которая сохраняет оценку по пяти шкалам (0–10) и комментарий LLM и
возвращает их как результат задачи. Шкалы: свежесть образа (`freshness`),
эмоциональная плотность (`emotional_density`), голос (`voice`), завершённость
(`completeness`) — от внешнего LLM-API; анти-повтор (`anti_repeat`) — вычисляется
из эмбеддингов «Забора» (этап 2). Flow аналогичен «Забору»: `POST` ставит задачу и
возвращает `task_id`, затем статус опрашивается через `GET /api/tasks/{task_id}/`.

**`POST /api/poems/{id}/assessment/`**

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | `int` | ID стиха |

### Ответ `202 Accepted`

```json
{
  "task_id": "3f1c2e9a-7b4d-4c8a-9e2f-1a0b2c3d4e5f"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `task_id` | `string` | UUID задачи Celery. Используйте для опроса `GET /api/tasks/{task_id}/`. |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `404` | `PoemNotFoundError` | Poem not found | Стих не найден или не принадлежит текущему автору |

### Опрос результата

Результат приходит в поле `result` ответа `GET /api/tasks/{task_id}/` (раздел 7) при
`status: "ready"`:

```json
{
  "status": "ready",
  "result": {
    "scores": {
      "freshness": 8,
      "emotional_density": 7,
      "voice": 6,
      "completeness": 9,
      "anti_repeat": 4
    },
    "comment": "Крепкий стих, но образ уже встречался.",
    "model_name": "meta-llama/llama-3.3-70b-instruct:free"
  }
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `result.scores` | `object` | Баллы по пяти шкалам (0–10). |
| `result.scores.freshness` | `int` \| `null` | Свежесть образа. |
| `result.scores.emotional_density` | `int` \| `null` | Эмоциональная плотность. |
| `result.scores.voice` | `int` \| `null` | Голос (узнаваемость интонации). |
| `result.scores.completeness` | `int` \| `null` | Завершённость. |
| `result.scores.anti_repeat` | `int` \| `null` | Анти-повтор: выше — дальше от уже написанного. `null`, пока нет эмбеддинга. |
| `result.comment` | `string` | Короткий комментарий LLM. |
| `result.model_name` | `string` | Какая LLM-модель посчитала оценку (для A/B). |

## 6. Запуск поиска похожих стихов

Ставит Celery-задачу сравнения в очередь. Сама задача (`compute_poem_similarity`)
ждёт готовности эмбеддинга (повторяет попытку с `countdown=2` до 15 раз, пока вектор
не посчитан воркером очереди `embeddings`), поэтому race condition при `PUT` исключён.
Возвращает `task_id` для опроса статуса (см. раздел 7).

**`POST /api/poems/{id}/similar/`**

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | `int` | ID стиха |

### Ответ `202 Accepted`

```json
{
  "task_id": "3f1c2e9a-7b4d-4c8a-9e2f-1a0b2c3d4e5f"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `task_id` | `string` | UUID задачи Celery. Используйте для опроса `GET /api/tasks/{task_id}/`. |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `404` | `PoemNotFoundError` | Poem not found | Стих не найден или не принадлежит текущему автору |

## 7. Статус асинхронной задачи

Позволяет опрашивать статус любой Celery-задачи по её UUID. Доступно любому
авторизованному автору; результаты не пересекают авторов (поиск «Забора» фильтруется
по `poem__author`).

**`GET /api/tasks/{task_id}/`**

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|----------|
| `task_id` | `string` | UUID задачи, полученный из `POST /api/poems/{id}/similar/` |

### Ответ `200 OK` (задача в работе)

```json
{
  "status": "pending"
}
```

### Ответ `200 OK` (задача завершена успешно)

```json
{
  "status": "ready",
  "result": [
    {
      "id": 12,
      "title": "* * *",
      "comment": "16.07.2026",
      "similarity": 0.91
    }
  ]
}
```

### Ответ `200 OK` (задача упала)

```json
{
  "status": "error",
  "message": "Embedding not ready"
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `status` | `string` | `pending` — задача ещё выполняется; `ready` — успешно, результат в `result`; `error` — задача упала, текст ошибки в `message`. |
| `result` | `array` | Список похожих стихов (тот же формат полей, что у `result[]` в разделе 5). |
| `result[].id` | `int` | ID похожего стиха |
| `result[].title` | `string` | Заголовок похожего стиха |
| `result[].comment` | `string` | Комментарий похожего стиха |
| `result[].similarity` | `float` | Косинусная близость, округлено до 4 знаков |
| `message` | `string` | Текст ошибки при `status: "error"` |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |

## Сводная таблица эндпоинтов

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/poems/` | Список стихов с пагинацией |
| `POST` | `/api/poems/` | Создание стиха (ставит эмбеддинг в очередь `embeddings`) |
| `GET` | `/api/poems/stats/` | Статистика для дашборда (включая средние баллы оценки) |

| `GET` | `/api/tasks/{task_id}/` | Статус асинхронной задачи (`pending`/`ready`/`error`) |

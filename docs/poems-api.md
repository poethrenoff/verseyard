# API Poems

## Общая информация

**Базовый путь:** `/api/poems/`

Все эндпоинты требуют JWT-токен в заголовке `Authorization: Bearer <token>`.

Объект стиха (`Poem`) имеет следующие поля:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | `int` | Идентификатор стиха |
| `title` | `string` | Заголовок. Если не задан явно, выводится из первой строки текста в формате `"<Первая строка>..."` |
| `content` | `string` | Текст стиха (нормализуется при сохранении) |
| `comment` | `string` | Комментарий. Если не задан, подставляется текущая дата в формате `ДД.ММ.ГГГГ` |

**Формат ошибок:**

```json
{
  "error": {
    "message": "Текст ошибки",
    "type": "ErrorType",
    "code": 400,
    "form_errors": {
      "field": [
        "Описание ошибки поля"
      ]
    }
  }
}
```

## 1. Список стихов

Возвращает список стихов текущего автора с пагинацией.

**`GET /api/poems/`**

### Параметры запроса (query params)

| Параметр | Тип | Обязательный | По умолчанию | Описание |
|----------|-----|--------------|--------------|----------|
| `collection` | `int` \| `null` | Нет | `null` | Фильтр по коллекции. `null` — стихи вне коллекций. |
| `query` | `string` | Нет | `""` | Поиск по вхождению подстроки в `title`, `content` или `comment`. |

### Ответ `200 OK`

```json
{
  "count": 42,
  "next": "http://host/api/poems/?page=2",
  "previous": null,
  "results": [
    {
      "id": 1,
      "title": "\"Первая строка...\"",
      "content": "текст",
      "comment": "16.07.2026"
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `count` | `int` | Общее количество стихов |
| `next` | `string` \| `null` | URL следующей страницы |
| `previous` | `string` \| `null` | URL предыдущей страницы |
| `results` | `array` | Массив стихов |
| `results[].id` | `int` | Идентификатор стиха |
| `results[].title` | `string` | Заголовок стиха |
| `results[].content` | `string` | Текст стиха |
| `results[].comment` | `string` | Комментарий |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `400` | `InvalidFormError` | Invalid form | Невалидные query-параметры |

## 2. Создание стиха

Создаёт новый стих со `active=True` и без коллекции (`collection=null`). Позиция (`position`) вычисляется как максимальная позиция среди стихов автора вне коллекций + 1.

**`POST /api/poems/`**

### Тело запроса

```json
{
  "content": "Текст стиха.\nВторая строка.",
  "title": "Необязательный заголовок",
  "comment": "Необязательный комментарий"
}
```

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| `content` | `string` | Да | Текст стиха. Пробелы/пустые края нормализуются. |
| `title` | `string` | Нет | Если пусто — выводится из первой строки `content`. |
| `comment` | `string` | Нет | Если пусто — подставляется текущая дата. |

### Ответ `201 Created`

```json
{
  "id": 1,
  "title": "\"Первая строка...\"",
  "content": "Текст стиха.\nВторая строка.",
  "comment": "16.07.2026"
}
```

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `400` | `InvalidFormError` | Invalid form | Невалидные или отсутствующие поля |

## 3. Статистика для дашборда

Статистика по активным стихам (`active=True`) текущего автора.

**`GET /api/poems/stats/`**

### Ответ `200 OK`

```json
{
  "in_collection": 12,
  "uncategorized": 30,
  "remaining": 99958
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `in_collection` | `int` | Количество активных стихов, привязанных к коллекции (`collection != null`). |
| `uncategorized` | `int` | Количество активных стихов вне коллекций (`collection = null`). |
| `remaining` | `int` | `max(0, 100000 - (in_collection + uncategorized))` — лимит 100 000 стихов. |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |

## 4. Обновление стиха

Полное обновление стиха. Доступно только автору стиха.

**`PUT /api/poems/{id}/`**

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | `int` | ID стиха |

### Тело запроса

Все поля необязательны (`partial=True`, сервер дополняет `title`/`comment` из `content`, если они не переданы):

```json
{
  "content": "Новый текст стиха.",
  "title": "Новый заголовок",
  "comment": "Новый комментарий"
}
```

| Поле | Тип | Обязательный | Описание |
|------|-----|--------------|----------|
| `content` | `string` | Нет | Если передан — нормализуется. `title`/`comment` пересчитываются, если не заданы явно. |
| `title` | `string` | Нет | Если пусто/не передан — выводится из первой строки `content`. |
| `comment` | `string` | Нет | Если пусто/не передан — подставляется текущая дата. |

### Ответ `200 OK`

```json
{
  "id": 1,
  "title": "Новый заголовок",
  "content": "Новый текст стиха.",
  "comment": "Новый комментарий"
}
```

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `400` | `InvalidFormError` | Invalid form | Невалидные или отсутствующие поля |
| `404` | `PoemNotFoundError` | Poem not found | Стих не найден или не принадлежит текущему автору |

## 5. Похожие стихи («Забор»)

Семантический антиповтор. Сравнивается эмбеддинг стиха (модель `BAAI/bge-m3`,
1024 измерения, косинусная близость по L2-нормализованным векторам в pgvector) со всеми
активными (`active=True`) стихами того же автора, кроме самого проверяемого.

Эндпоинт неблокирующий: фронт опрашивает его после создания/обновления стиха, пока
эмбеддинг не посчитан воркером очереди `embeddings`.

**`GET /api/poems/{id}/similar/`**

### Параметры пути

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | `int` | ID стиха |

### Ответ `200 OK` (эмбеддинг ещё не готов)

```json
{
  "status": "pending",
  "similar": []
}
```

### Ответ `200 OK` (эмбеддинг готов)

```json
{
  "status": "ready",
  "similar": [
    {
      "id": 12,
      "title": "* * *",
      "comment": "16.07.2026",
      "similarity": 0.91
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `status` | `string` | `pending` — эмбеддинг ещё не посчитан; `ready` — эмбеддинг готов, поиск выполнен. |
| `similar` | `array` | Список похожих стихов (топ-`FENCE_SIMILARITY_LIMIT`, по умолчанию 5). |
| `similar[].id` | `int` | ID похожего стиха |
| `similar[].title` | `string` | Заголовок похожего стиха |
| `similar[].comment` | `string` | Комментарий похожего стиха |
| `similar[].similarity` | `float` | Косинусная близость (`1 - distance`), округлено до 4 знаков. Только значения ≥ `FENCE_SIMILARITY_THRESHOLD` (по умолчанию `0.8`). |

### Ошибки

| Код | Type | Сообщение | Условие |
|-----|------|-----------|---------|
| `401` | — | — | Не авторизован |
| `404` | `PoemNotFoundError` | Poem not found | Стих не найден или не принадлежит текущему автору |

> Примечание: при `POST`/`PUT` эмбеддинг ставится в очередь `embeddings` асинхронно.
> Пока воркер не посчитает вектор, эндпоинт возвращает `status: "pending"`.
